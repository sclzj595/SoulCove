## 当前版本状态

**版本**：V1.9（2026-07）
**构建**：Debug，scIDE 编译链接通过
**LSP**：clangd 19.1.0 集成完成，支持 C/C++ 项目内跳转、补全、悬停预览
**已知技术债**：原 44 项优化点（O1-O44）已在 V1.9 全链路增强中清理完毕；本次迭代过程中新增 8 项 API 兼容性 / 构建配置修复（详见 V1.9 记录「编译修复」分组）

## 三、迭代流程规范

### 3.1 迭代周期

- **小迭代**：1-3 天，处理 3-5 个优化点（适合第一、二批）
- **中迭代**：1 周，处理一个完整模块的优化（适合第三、四批）
- **大迭代**：2-3 周，涉及构建系统或架构调整（适合第五批）

### 3.2 迭代步骤

每个优化点（O 编号）的处理流程：

1. **立项**
   - 在本文档对应批次表中标记状态为「进行中」
   - 评估影响范围（是否影响三产品、是否破坏现有功能）

2. **编码**
   - 遵循现有编码规范（C++17 / Qt 信号槽 / 中文注释）
   - 修改前先 `Read` 相关文件理解上下文
   - 优先 `Edit` 现有文件，避免新建文件
   - 不添加多余的错误处理 / 注释 / 类型标注

3. **编译验证**
   - `cmake --build build --config Release`
   - 确保三产品（SoulCove IDE / SoulCove Notebook / SoulCove Notebook Lite）全部编译通过
   - 编译失败时优先修复，不引入 `#pragma` 等绕过手段

4. **功能验证**
   - 启动 scIDE，手动验证修改点
   - 检查 LSP 日志（`debug.log`）无异常
   - 验证不影响其他功能（回归测试）

5. **文档更新**
   - 在本文档「迭代历程」追加记录
   - 更新优化点状态为「已完成」
   - 如涉及用户可见变化，更新 `README.md`

6. **提交**（仅在用户明确要求时）
   - Git commit message 格式：`[O编号] 模块: 简述`
   - 示例：`[O1] LSP: 响应路由改为按请求类型独立字段`

### 3.3 状态标记约定

在本文档表格中使用以下状态标记（追加到「优化建议」列或新增「状态」列）：

- `待处理` — 未开始
- `进行中` — 编码中
- `待验证` — 编码完成，待编译/功能验证
- `已完成` — 验证通过
- `已搁置` — 暂不处理（需说明原因）

### 3.4 批次交付检查清单

每批次完成前确认：

- [ ] 批次内所有优化点状态为「已完成」或「已搁置」
- [ ] `cmake --build build --config Release` 三产品编译通过
- [ ] scIDE 启动正常，无崩溃
- [ ] LSP 功能正常（打开 C++ 项目，补全/跳转/hover 可用）
- [ ] `debug.log` 无新增 ERROR 级别日志
- [ ] 本文档「迭代历程」已更新
- [ ] 涉及用户可见变化时 `README.md` 已更新

开发新功能时发现的技术债：

1. 在本文档「优化路线图」对应批次追加新行
2. 编号延续（O45, O46, ...）
3. 评估优先级，归入合适批次
4. 不阻塞当前功能开发，记录后继续

### 5.3 技术债清理原则

- **不破坏现有功能**：优化前评估影响范围
- **小步快跑**：每次只处理 1-3 个优化点，频繁编译验证
- **先验证后优化**：优先处理有用户反馈的问题
- **避免过度工程**：只做必要修改，不引入不必要的抽象

### 6.3 常见问题

**Q: clangd 找不到 compile_commands.json？**
A: 确认 CMake 配置了 `set(CMAKE_EXPORT_COMPILE_COMMANDS ON)`，且 `build/compile_commands.json` 已生成。程序会自动向上查找。

**Q: Ctrl+Click 项目内头文件无法跳转？**
A: 检查 `LspManager::inferProjectRoot()` 是否正确推断项目根目录，`compile_commands.json` 是否在该目录或其 `build/` 子目录。

**Q: hover 弹窗频繁闪现？**
A: 已在 H1 修复（鼠标移动时立即隐藏弹窗 + 取消 pending 请求）。如仍出现，检查 `MyTextEdit::mouseMoveEvent` 是否发射 `hoverAborted`。

**Q: 主题切换后部分控件样式不刷新？**
A: 检查该控件是否连接了 `ThemeManager::themeChanged` 信号（参考 J3 修复）。

**Q: 切换标签页后 LSP 功能失效？**
A: 检查 `IdleTabTracker` 是否正确释放/恢复标签，`didClose` / `didOpen` 时序是否正确（参考 L3 修复）。
